Os trasladamos las aclaraciones solicitadas:
Q_007
Confirmamos que cada usuario dispone de un identificador único, personal e intransferible para acceder a la plataforma. La unicidad se garantiza mediante el identificador interno de usuario y el correo electrónico asociado a cada cuenta. No se utilizan cuentas compartidas para usuarios nominativos.
Q_008
La identificación y gestión de usuarios con roles de administración se realiza mediante un modelo de permisos basado en perfiles. Los roles administrativos se asignan únicamente a usuarios autorizados y se revisan periódicamente. La plataforma permite identificar qué usuarios tienen privilegios de administración y mantiene registro de los cambios realizados sobre dichos permisos.
Q_011
El repositorio de usuarios utilizado actualmente es una base de datos local propia de la aplicación. No existe integración activa con LDAP, Active Directory, IDP corporativo ni con un sistema IAM externo para la gestión centralizada de identidades.
Q_012
Existe un procedimiento interno documentado para la gestión del ciclo de vida de usuarios. Las altas se realizan previa solicitud autorizada, las modificaciones de permisos requieren aprobación del responsable correspondiente y las bajas se ejecutan tras la notificación de finalización de acceso. Los cambios son registrados y ejecutados por usuarios con privilegios administrativos. El plazo habitual de ejecución de bajas es inferior a 48 horas laborables desde la recepción de la solicitud validada.
Q_013
La plataforma implementa control de acceso basado en roles. Existen roles predefinidos, como usuario estándar, supervisor y administrador. Los permisos se asignan en función del perfil del usuario y del alcance funcional requerido. La granularidad permite limitar el acceso a funcionalidades de administración, consulta y operación según el rol asignado.
Q_015
El sistema aplica una política de contraseñas robusta para usuarios nominativos. La política contempla longitud mínima, complejidad de caracteres, bloqueo tras intentos fallidos y restricciones para evitar contraseñas débiles. El uso de cadenas aleatorias de 128 bits aplica únicamente a credenciales técnicas o tokens generados por la plataforma, no sustituye a la política de contraseñas de usuarios.
Q_019
Las contraseñas no se almacenan en claro ni mediante cifrado reversible. Se almacenan mediante hash seguro utilizando bcrypt, con salt único por usuario y factor de coste configurado conforme a las recomendaciones actuales de la plataforma.
Q_021
El tiempo de expiración de sesión está configurado actualmente en 60 minutos de inactividad, pero es configurable por política de la plataforma. Puede reducirse a valores inferiores, como 30 minutos, para ajustarse a los requisitos de seguridad del cliente. La política aplica a todos los usuarios autenticados, incluidos usuarios estándar y administradores.
Q_023
Para las comunicaciones HTTPS se soportan TLS 1.2 y TLS 1.3. Las versiones SSLv2, SSLv3, TLS 1.0 y TLS 1.1 se encuentran deshabilitadas en los endpoints expuestos del servicio.
Q_028
Los backups se cifran mediante AES-256. El cifrado aplica a las copias de seguridad almacenadas en el entorno de infraestructura. La gestión de claves se realiza mediante mecanismos del proveedor cloud, con acceso restringido a personal autorizado.
Q_029
Confirmamos que las comunicaciones se realizan mediante HTTPS con TLS 1.2 o superior. Las versiones obsoletas SSL, TLS 1.0 y TLS 1.1 están deshabilitadas. No se permite tráfico HTTP sin cifrar para el acceso a la aplicación.
Q_030
El proceso de backup se ejecuta de forma automatizada sobre la base de datos y los volúmenes persistentes asociados al servicio. Las copias se almacenan cifradas en infraestructura cloud ubicada en la Unión Europea. El periodo de retención actual es de 30 días. La integridad de las copias se verifica mediante controles del servicio de almacenamiento y se realizan pruebas periódicas de restauración para validar la recuperabilidad de los datos.
Q_031
Los valores definidos para este servicio son los siguientes: RTO de 8 horas y RPO de 24 horas. Estos valores aplican al servicio SaaS en condiciones normales de recuperación ante una incidencia técnica relevante.
Q_032
La política de retención de datos está actualmente en proceso de formalización y todavía no se encuentra aprobada de forma definitiva. Aunque el periodo operativo utilizado para copias de seguridad es de 30 días, la política documental completa con todos los criterios de conservación, borrado y acceso está pendiente de validación interna.
Q_034
Sí, existe un proceso documentado para la finalización y baja del servicio. El procedimiento contempla la notificación de baja, validación con el cliente, exportación de datos si aplica, revocación de accesos, eliminación lógica del servicio y eliminación de datos conforme a la política de retención y ciclo de vida de backups.
Q_036
La devolución de la información se realiza mediante exportación en formato estructurado, normalmente CSV o XLSX para datos tabulares y ZIP para conjuntos documentales. La integridad de la exportación se valida mediante comprobaciones de consistencia y, si se requiere, mediante hash del paquete exportado. El plazo habitual para la entrega de la exportación es de hasta 10 días laborables desde la solicitud validada de finalización del servicio.
Q_045
Los entornos de desarrollo y pruebas están lógicamente separados del entorno de producción. Cuentan con bases de datos, credenciales y accesos diferenciados. No se utilizan datos reales de producción en desarrollo o pruebas salvo autorización expresa y, en ese caso, se aplican medidas de anonimización o seudonimización. Los accesos a cada entorno se gestionan de forma independiente.
Q_057
Existe un procedimiento formalizado de comunicación y escalado de incidentes de seguridad. El procedimiento define canales de notificación, responsables internos, criterios de severidad, tiempos de escalado y comunicación al cliente cuando el incidente pueda afectar a la confidencialidad, integridad o disponibilidad del servicio. Los incidentes se registran y se realiza seguimiento hasta su cierre.
